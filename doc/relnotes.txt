Версия 0.8
 В архив с программой добавлены файлы File_id.diz и Descript.ion
с кратким описанием архива.

 Добавлена поддержка масок в расширениях файлов.
 В атрибуте 'extention' поддерживается задание маски имени файла.
Символ '*' обозначает любое число любых символов, символ '?' - однократное 
вхождение любого символа. Символ '*' можно использовать как в конце, так 
и в середине маски (последняя ситуация не тестировалась исчерпывающе и может 
содержать ошибки в реализации). Ниже приведено несколько тестовых случаев,
на которых проверялась корректность сравнения по маске:

+ .c*p equals to .cxxp
+ .in*mat* equals to .information
+ .in*mat* equals to .inmat
- .in*mat* not equal to .infwat
+ .in*?at* equals to .infwat
- .cpp not equal to .cxx
- .c?? not equal to .hpp
+ .c?? equals to .cpp
+ .c?? equals to .cxx
- .c* not equal to .hpp
+ .c* equals to .cpp
+ .c* equals to .cxx
+ .c* equals to .c



Версия 0.9
 Изменен алгоритм поиска файла с пользовательскими настройками.
Раньше поиск файла 'texttp-custom-info.xml' производился только в 
том каталоге, где расположен редактируемый файл.
Теперь поиск файла 'texttp-custom-info.xml' производиться сначала в том же
каталоге, где находится и редактируемый файл, затем во всех каталогах 
выше, вплоть до корневого каталога. Поиск прекращается, когда файл
найден в одном из этих каталогов.

 Далее по мелочам.

 Добавлены шорткаты i1pack-i8pack, ipoppack, вставляющие
    #include <pshpack1.h>
    ...
    #include <pshpack8.h>
    #include <poppack.h>

 Это совсем к данному плагину не относиться, но пусть будет тут.

 В архив с программой добавлен файл CharCase.reg.
Он создает клавиатурные макросы для изменения регистра символов.
Сочетания клавиш соответствуют аналогичным в среде Visual Studio.
    Ctrl+Shift+U - UPPERCASE
    Ctrl+U       - lowercase
    Ctrl+Alt+U   - Each Word Begins With Uppercase Char (в студии такого шортката нет)
Для корректной работы макросов необходим плагин "EditCase", в меню плагинов (F11)
ему должна быть назначена горячая клавиша 'c'.



Версия 0.901
 Пре-релиз. Не реализованы функции "Раскоментировать выделенное",
"Выделить закоментированное", см. далее.

Можно распечатать и повесить рядом.

 Введены некоторые проверки для конфигурационных файлов.
Часть атрибутов сделана обязательными - их отсутствие вызовет ошибку 
загрузки конфигурационного файла.
 Обязятельные атрибуты:
  тэг file-type - type, extention
  тэг template  - name
  тэг comment   - file-type
  тэг shortcut  - name, file-type

 Описание типа файла csrc, ранее включавшее файлы с расширениями 
".c,.cpp,.cxx,.c++,.h,.hpp" разделено на два типа c-src с расширениями 
".c,.h" и cpp-src с расширениями ".cpp,.cxx,.c++,.hpp".
 Стало возможным задавать одинаковый шорткат одновременно для нескольких
типов файлов путем перечисления типов файлов через запятую
(ранее для каждого из типов следовало продублировать строку - описание шортката)
 Все описания шорткатов изменены так, чтобы работать для файлов c-src и cpp-src
(все шорткаты должны работать так же, как и раньше, за исключением шорткатов
"//", "/*" и "separator")
 Подробнее см. раздел "Шорткаты" в файле "readme.txt".

 !!! Значительное расширение функциональности !!!

Ctrl+Alt+Z  Уменьшить отступ         -> Shift+Tab
Ctrl+X      Увеличить отступ (1 пр)  -> Ctrl+Space

(плагинов третьего уровня). В поставку включены два модуля форматирования - 
форматирование простого текста и вызов внешних команд для форматирования.
 Включен в поставку заголовочный файл с описанием интерфейса плагинов. Также включены
исходные тексты плагинов.
